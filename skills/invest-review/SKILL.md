# Invest Review Skill

## 核心职责

Agent 的自我审计与进化引擎。系统性回顾认知积累，推动投资原则、能力圈和关注清单的理性进化。

## 触发条件

| 触发方式 | 场景 |
|---|---|
| invest-analysis 建议 | 分析中发现当前判断与 SOUL.md 原则冲突 |
| 用户主动请求 | "帮我复盘一下投资原则" |
| 定期自动 | 每月轻度复核，每季深度复核 |
| 重大事件驱动 | 持仓暴跌 30%+、重大黑天鹅 |

## 数据源

| 数据源 | 路径 | 用途 |
|---|---|---|
| 投资灵魂 | `SOUL.md` | 被审查对象 |
| 个股分析 | `views/*.md` | 近期判断记录 |
| 长期摘要 | `MEMORY.md` | 历史决策脉络 |
| 日记流水 | `memory/` | 近期偏好信号 |
| 行业认知 | `domains/*.md` | 行业理解框架 |
| 公司资料 | `{OPENCLAW_DATA_DIR}/companies/` | 交叉验证证据 |

## 执行步骤

1. 读取 `references/review-checklist.md` 获取 4 维度检查清单
2. **轻度复核**（每月）：只做维度 1 锚点检查 + 维度 3 关注清单快检
3. **深度复核**（每季）：4 个维度全覆盖，含判断准确度回溯
4. 对照 views/、memory/、domains/ 中的实际数据逐项审查
5. 生成复核报告

## 输出格式

### 1. 健康度总评

| 维度 | 状态 | 说明 |
|---|---|---|
| 锚点字段 | 健康 / 需调整 / 过时 | 每个锚点简评 |
| 能力圈 | 聚焦 / 扩展中 / 模糊 | 边界是否清晰 |
| 关注清单 | 活跃 / 冷却 / 臃肿 | 标的是否得到持续关注 |
| 判断准确度 | 高 / 中 / 需反思 | 基于待验证事项的回溯 |

### 2. 建议修改项

每条包含：修改目标、建议内容、证据链（≥2条）、风险评估、信心等级（强烈建议/值得考虑/仅供参考）

### 3. 认知进化建议

哪些认知应沉淀到 SOUL.md？哪些行业理解应更新到 domains/？

## 硬约束

1. **不自动修改 SOUL.md**：所有修改建议必须用户确认后才生效
2. 不做投资分析，只检查原则和认知体系的一致性
3. 每条建议必须引用具体的分析记录或事实
4. 保持审慎——倾向于不修改，除非有强有力的证据
