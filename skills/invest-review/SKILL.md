# Invest Review Skill

## 核心职责

`invest-review` 是 Agent 的**自我审计与进化引擎**。它不只是检查 SOUL.md 是否过时，而是系统性地回顾 Agent 的认知积累，推动投资原则、能力圈和关注清单的理性进化。

把它理解为"投资者的年度体检"——不是看病，而是检查整体健康状况。

## 触发条件

| 触发方式 | 场景 |
|---|---|
| invest-analysis 建议 | 分析过程中发现当前判断与 SOUL.md 原则冲突 |
| 用户主动请求 | "帮我复盘一下我的投资原则" |
| 定期自动 | 建议每月一次轻度复核，每季一次深度复核 |
| 重大事件驱动 | 持仓出现重大变故（如暴跌 30%+、重大黑天鹅） |

## 数据源

| 数据源 | 路径 | 用途 |
|---|---|---|
| 投资灵魂 | `SOUL.md` | 当前锚点字段（被审查对象） |
| 个股分析 | `views/*.md` | 近期所有公司的分析积累和判断记录 |
| 长期摘要 | `MEMORY.md` | 历史决策脉络和重大认知转折点 |
| 日记流水 | `memory/` | 近期对话细节和偏好信号 |
| 行业认知 | `domains/*.md` | 当前的行业理解框架 |
| 公司资料 | `data/companies/` | 用于交叉验证判断的原始证据 |

## 复核框架

### 维度 1：锚点字段校验

逐条检查 SOUL.md 中每个锚点字段是否仍然成立：

**核心信仰 (Core Beliefs)**
- 过去一个周期内，是否有分析结论与核心信仰产生过冲突？
- 这些冲突是"噪声"（一次性事件）还是"信号"（趋势性变化）？
- 核心信仰是否过于笼统以至于无法指导实际判断？

**投资风格**
- 回顾 views/ 中的实际分析，你的行为是否与声称的风格一致？
  - 说"长期持有"——但是否在短期波动中频繁更新 views/ 的结论？
  - 说"集中持仓"——但关注清单是否越来越长？
  - 说"偏好高护城河"——但是否在分析低护城河公司时同样给出看多结论？

**风险边界**
- 设定的风险边界是否经受过实际考验？
- 如果没被考验过，是因为市场没给机会，还是因为边界设得太宽？
- 回顾历史上触及风险边界的情况，当时的决策是否正确？

**核心逻辑要求**
- 所有 views/ 中的"看多"结论是否都通过了核心逻辑的双重验证？
- 如果有绕过核心逻辑的例外，原因是什么？例外是否合理？

**禁忌 (Taboos)**
- 是否有分析行为违反了禁忌规则？（如声称不做技术面分析，但在 views/ 中出现了 K 线相关判断）

### 维度 2：能力圈审计

**能力圈边界检查**
- SOUL.md 声明的能力圈范围 vs views/ 中实际分析过的行业，是否匹配？
- 如果分析了能力圈外的公司：
  - 分析质量如何？（与能力圈内的分析对比深度和准确度）
  - 是否应该扩展能力圈？（有充分的学习积累）
  - 还是应该标记为"越界"并收缩？

**能力圈深度检查**
- 在声称的能力圈内，domains/{domain}.md 的认知深度是否在增长？
- 是否有行业出现了结构性变化，导致之前的行业认知需要更新？

### 维度 3：关注清单复核

**核心重仓 (Core)**
- 每只 Core 标的是否有持续更新的 views/？
- 如果某只 Core 标的超过 1 个月没有新的分析更新，是否应该降级到 Monitoring？
- Core 标的之间是否存在风险集中（同行业、同逻辑、同催化剂）？

**持续观察 (Monitoring)**
- Monitoring 中是否有标的，其认知积累已经足够升级到 Core？
- 是否有标的已经失去关注价值（基本面恶化、超出能力圈），应该移除？

**备选池 (Potential)**
- 是否有在日常新闻/分析中反复出现但不在清单中的公司？
- 是否应该主动扩展雷达范围？

### 维度 4：判断准确度回溯

**预测验证**
- 回顾 views/ 中过去的"待验证事项"，已经可以验证的有哪些？
- 验证结果：证实了多少？证伪了多少？
- 证伪的判断中，错误的根因是什么？（信息不足？逻辑错误？情绪偏差？）

**认知偏差检测**
- 是否存在"锚定效应"？（对早期判断过于坚持，忽视新证据）
- 是否存在"确认偏差"？（只关注支持现有结论的信息）
- 是否存在"损失厌恶"？（对已看多的公司不愿承认看错）

## 输出

复核结论报告，包含以下部分：

### 1. 健康度总评

对 SOUL.md 的整体健康状态打分（仅作为讨论锚点，不追求精确）：

| 维度 | 状态 | 说明 |
|---|---|---|
| 锚点字段 | 健康 / 需调整 / 过时 | 每个锚点的简评 |
| 能力圈 | 聚焦 / 扩展中 / 模糊 | 边界是否清晰 |
| 关注清单 | 活跃 / 冷却 / 臃肿 | 标的是否得到持续关注 |
| 判断准确度 | 高 / 中 / 需反思 | 基于待验证事项的回溯 |

### 2. 建议修改项

每条建议包含：
- **修改目标**：SOUL.md 的哪个字段 / 关注清单的哪个调整
- **建议内容**：具体的新值或调整方向
- **证据链**：至少 2 条来自 views/、data/companies/ 或 domains/ 的支撑证据
- **风险评估**：修改后可能带来的风险
- **信心等级**：强烈建议 / 值得考虑 / 仅供参考

### 3. 认知进化建议

- 哪些新发现的认知应该沉淀到 `SOUL.md`？
- 哪些行业理解应该更新到 `domains/{domain}.md`？
- 是否有分析方法论层面的改进？

## 执行规则

- **不自动修改 SOUL.md**：所有修改建议必须用户确认后才生效
- **不做投资分析**：复核只检查原则和认知体系的一致性，不产出新的个股判断
- **必须有证据支撑**：每条建议必须引用具体的分析记录或事实
- **保持审慎**：倾向于不修改，除非有强有力的证据表明原则需要进化
- **区分"轻度复核"和"深度复核"**：
  - 轻度（每月）：只做维度 1 的锚点检查 + 维度 3 的关注清单快检
  - 深度（每季）：四个维度全部覆盖，含判断准确度回溯
