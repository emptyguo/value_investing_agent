# Invest Analysis Skill

## 核心职责

`invest-analysis` 是每个 Agent 的**个人化深度分析技能**。它不是套模板填空，而是将用户的投资认知（SOUL.md）与多种数据源结合，形成有判断力的分析。

分析的深度和方向由用户的 SOUL.md 决定——同样一份年报，不同用户的 Agent 应产出不同侧重的分析。

## 数据源

分析时综合以下数据源，按信息质量排序：

| 数据源 | 路径 | 可信度 | 说明 |
|---|---|---|---|
| 年报/季报/公告 | `data/companies/{co}/filings/` | L1 最高 | 官方披露，可直接作为事实引用 |
| 业绩会/路演 | `data/companies/{co}/transcripts/` | L2 高 | 管理层口径，注意区分事实与展望 |
| 卖方研报 | `data/companies/{co}/sellside/` | L3 中 | 有参考价值但有利益偏差，重数据轻结论 |
| 专家访谈/渠道调研 | `data/companies/{co}/primary/` | L4 因源而异 | 一手信息，需交叉验证 |
| 传闻/内部信息 | `data/companies/{co}/unofficial/` | L5 低 | 仅作为线索，不可直接作为判断依据 |
| 新闻 | `data/companies/{co}/news/` | L6 需核实 | 时效性强，但噪音大 |
| 行业认知 | `domains/{domain}.md` | 背景 | 行业共性规律 |
| 个人历史判断 | `views/{company}.md` | 对照 | 检验之前的判断是否被证实/证伪 |
| 投资原则 | `SOUL.md` | 框架 | 决定分析的方向和标准 |

**注意**：如果 PDF 文件尚未解析为 `.md`，先调用 `invest-pdf-parser` 解析后再读取。

## 分析框架

以下维度不是必须全部覆盖的清单，而是 Agent 根据 SOUL.md 和当前语境**选择性深入**的方向。

### 定性维度

**商业模式与护城河**
- 公司靠什么赚钱？收入结构是否健康？
- 护城河是什么类型（品牌、网络效应、转换成本、规模经济、牌照/专利）？
- 护城河是在加深还是被侵蚀？证据是什么？

**竞争格局**
- 市场集中度如何？公司的市场地位变化趋势？
- 竞品的动态对公司构成什么威胁或机会？
- 行业是存量博弈还是增量市场？

**管理层评估**
- 管理层的资本配置能力如何？（看回购、分红、并购记录）
- 管理层说的和做的是否一致？（对比历史电话会承诺与实际执行）
- 有无治理风险？（股权结构、关联交易、管理层变动）

**催化剂与时间节点**
- 近期有什么可能改变公司价值的事件？
- 市场预期和实际情况的差距在哪里？

### 定量维度

**盈利能力**
- 毛利率、净利率趋势（至少看 3 年）
- ROE/ROIC 水平及变化方向
- 自由现金流与净利润的关系（现金利润比）

**成长性**
- 营收增速、利润增速趋势
- 增长是内生的还是靠并购？
- 增长的可持续性判断

**财务健康**
- 资产负债结构（负债率、有息负债比例）
- 现金流状况（经营性现金流是否覆盖资本开支）
- 应收账款/存货周转变化

**估值**
- 当前估值水平（PE、PB、EV/EBITDA 等，选适合该行业的指标）
- 与历史估值区间对比
- 与同业可比公司对比
- 如果 SOUL.md 中有明确的估值标准（如 DCF、安全边际要求），按其标准计算

## 输出

分析结果写入 workspace 的 `views/{company}.md`，格式不固定，但必须包含：

1. **当前观点**：一句话结论（看多/看空/中立 + 核心理由）
2. **关键判断与证据**：每个判断必须引用具体数据源
3. **与上次分析的对比**：哪些判断被证实了？哪些需要修正？
4. **风险与证伪条件**：什么情况下当前判断会被推翻？
5. **待验证事项**：需要继续追踪的 open questions
6. **更新记录**：`[YYYY-MM-DD] 简述本次分析的触发原因和核心结论`

## 认知沉淀

在分析过程中，如果发现了可复用的投资认知（不限于当前公司），记录到 `memory/YYYY-MM-DD.md`：

- 对某类商业模式的新理解
- 对某个行业规律的新发现
- 对自己分析方法的反思

当某个认知在多次分析中反复得到验证，Agent 应提议将其沉淀到 `SOUL.md`（走原则确认流程）或 `domains/{domain}.md`（行业共识无需确认）。

## 执行规则

- **分析必须锚定 SOUL.md**：如果 SOUL.md 为空，用通用价值投资框架但标注"尚未形成个性化原则"
- **事实与观点严格分离**：数据是数据，判断是判断，不混淆
- **承认不确定性**：信息不足时明确标注"信息不足，暂缓判断"，不凑结论
- **不抓新闻**（那是 invest-news 的职责）
- **不改 SOUL.md**（那需要 invest-review + 用户确认）
- **不做买卖建议**：只提供分析框架和证据梳理，不替用户决策
