# Invest Digest Skill

## 核心职责

`invest-digest` 是每日新闻策展技能，将 invest-news 采集的原始新闻加工为**可直接阅读的个性化新闻简报**。

它连接了"原始采集"和"深度分析"之间的缺口：

```
invest-news（采集原始新闻）
  → invest-ingest（按公司分流）
  → invest-digest（筛选、核实、排序、成简报） ← 本技能
  → invest-analysis（深度分析）
```

## 数据源

| 输入 | 路径 | 用途 |
|---|---|---|
| 当日原始新闻 | `~/.openclaw/workspace/data/news/raw/YYYY-MM-DD.jsonl` | 主要输入 |
| 公司分流新闻 | `~/.openclaw/workspace/data/companies/{co}/news/raw/` | 补充细节 |
| 关注清单 | `SOUL.md` → Active Watchlist | 决定哪些公司的新闻需要关注 |
| 公司配置 | `~/.openclaw/workspace/data/references/companies.json` | 竞品和行业关键词关系 |
| 行业认知 | `domains/{domain}.md` | 辅助判断行业新闻的重要性 |

## 执行流程

### Step 1：范围确定

根据 SOUL.md 的 Active Watchlist 确定今日需要关注的公司列表。对每家公司，同时关注：

- **个股新闻**：直接提及该公司的新闻
- **竞品新闻**：companies.json 中该公司的 competitors 动态
- **行业新闻**：该公司所属行业的宏观动态（来自 industry_keywords）

### Step 2：信息核实

对每条新闻进行真实性初筛：

**多源交叉验证**

- 同一事件是否有 2 个以上独立信源报道？
- 信源是否为可信媒体（官方公告 > 主流财经媒体 > 自媒体/传闻）？
- 如只有单一信源且非官方渠道，标记为`[待核实]`

**核实等级**

| 等级 | 条件 | 标记 |
|---|---|---|
| 已确认 | 官方公告或多个独立信源 | 无标记 |
| 基本可信 | 主流财经媒体单独报道 | 无标记 |
| 待核实 | 仅自媒体/小道消息/单一信源 | `[待核实]` |
| 存疑 | 信源不明或内容矛盾 | `[存疑]` 并附原因 |

**不删除任何新闻**——即使存疑也保留并标记，因为有些小道消息后来会被证实。

### Step 3：重要性排序

按以下维度综合打分，排序后输出：

**高重要性**（影响投资判断）

- 业绩发布（年报、季报、业绩预告）
- 重大资本运作（并购、增发、回购、分拆上市）
- 管理层变动（CEO/CFO 更换、核心高管离职）
- 监管政策变化（行业新规、反垄断、税收政策）
- 重大竞争格局变化（新进入者、价格战、市场份额大幅变动）

**中重要性**（值得关注但不急）

- 产品/业务动态（新产品发布、业务扩张/收缩）
- 行业数据更新（月度销售数据、行业协会报告）
- 竞品的重要动态
- 券商评级变化

**低重要性**（存档即可）

- 日常运营新闻
- 重复报道同一事件的不同角度
- 行业一般性新闻

排序时还要考虑**用户的个人视角**：如果 SOUL.md 锚点中特别看重某个维度（如"核心逻辑要求：自由现金流增长"），与该维度相关的新闻应提升优先级。

### Step 4：生成简报

输出格式（按公司分组，组内按重要性排序）：

```markdown
# 每日投资简报 YYYY-MM-DD

## 核心重仓

### 腾讯控股 (00700)
1. **[高]** 腾讯2025年报发布：净利润同比增长XX%
   - 来源：腾讯控股公告 | [链接](https://...)
   - 要点：（2-3句话概括关键数据）

2. **[中]** 视频号商业化进展：Q4广告收入环比增长XX%
   - 来源：财联社 | [链接](https://...)
   - 要点：...

### 腾讯音乐 (01698)
1. **[中]** 网易云音乐宣布独家版权到期不续约 [待核实]
   - 来源：某自媒体 | [链接](https://...)
   - 要点：...
   - 核实说明：仅单一自媒体报道，官方未确认

## 持续观察

### 美团 (03690)
...

## 行业动态
- **互联网**：...
- **家电**：...

## 今日无重要更新
- 茅台 (600519)
- 拼多多 (PDD)
```

## 输出位置

- **主要输出**：直接在对话中呈现给用户
- **存档**：写入 `memory/YYYY-MM-DD.md` 的当日记录中，标题为 `## 每日简报`
- **推送**（可选）：如用户配置了定时推送，通过飞书消息发送

## 执行规则

- **必须带链接**：每条新闻附原始链接，用户可以点击查看原文
- **不做投资建议**：只呈现事实和重要性判断，不输出"建议买入/卖出"
- **不做深度分析**：那是 invest-analysis 的职责，简报只做"发生了什么、有多重要"
- **空结果也要报告**：如果今日无重要新闻，明确告知"今日关注清单中的标的无重要更新"
- **可由 cron 每日自动触发**：建议在 invest-news + invest-ingest 执行完成后运行
